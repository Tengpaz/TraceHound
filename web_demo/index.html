DATA PROCESSING

数据处理方法、流程、结果

从官方训练集到可用于 Reason 分析的高质量数据

初始数据结构

  • 训练数据包含基础二分类任务数据。
  • 训练数据包含三因素细粒度分类任务数据。
  • 评测数据额外包含 Label、三因素标签、Reason 和 Source。

评测字段含义

Label
Safe / Unsafe 分别以 0 / 1 表示。
Reason
解释模型分类理由，用于分类+理由分析任务。
Source
描述风险来源，包括 Safe、Unsafe、Benign、False_Refusal。
Safe 轨迹有风险，但模型正确防御。
Unsafe 轨迹有风险，且模型未采取任何防御手段。
Benign 轨迹本身无风险。
False_Refusal 模型尝试防御但未成功，最终造成风险执行。
01

保留高质量官方数据

优先使用官方二分类训练集中的 Safe 部分和三因素细粒度训练集中的 Unsafe 部分，降低 LLM 标注噪声。

02

补齐 Reason 与三因素标签

对 Safe 轨迹补充三因素+Reason；对 Unsafe 细粒度轨迹补充 Reason。Safe 的三因素标注相对简单，标注风险更低。

03

筛选合并训练集

抛弃二分类训练集中的 Unsafe 轨迹，直接合并细粒度 Unsafe 数据，形成更完整的三因素+Reason 数据集。

04

清洗格式与重复样本

统一 JSON 输出 Prompt，规范三因素标签大小写和间隔符，并处理二分类数据中约 2000 条重复数据。

METHOD

方法介绍

用官方训练数据微调小模型，提高特定评测任务效果

二分类安全判定

以 Qwen3.5-0.8B 为基座，对基础 Safe / Unsafe 分类任务分别进行全参数 SFT 与 LoRA 微调，并比较准确率、F1、格式无效率和输出长度。

三因素细粒度分类

围绕 Risk Source、Failure Mode、Harm Type 三个维度进行全参数 SFT，使模型能定位轨迹危险因素，而不只给出粗粒度安全标签。

Reason 分析增强

在数据构造阶段补齐 Reason 字段，让模型学习分类理由与风险证据之间的对应关系，同时通过清洗和筛选降低标注误差。

GRPO 强化优化

针对三因素+Reason 推理任务开展 GRPO 强化学习训练，奖励侧重 JSON 有效性、三因素一致性和 Reason 证据对齐。

EXPERIMENT RESULTS

实验结果展示分析

SFT 显著改善分类准确性和 JSON 格式稳定性

二分类任务：全参数 SFT 优于 LoRA

全参数 SFT 将准确率从 0.5185 提升到 0.7303，F1 从 0.6829 提升到 0.7791；LoRA 有提升但幅度有限，且出现 0.0067 的无效率。

模型 准确率 F1 无效率 Output Tokens mean / max / min / median
Qwen3.5-0.8B 0.5185 0.6829 0 13 / 13 / 13 / 13
TraceHound-Basev1 0.7303 0.7791 0 13 / 13 / 13 / 13
TraceHound-LoRAv1 0.6767 0.7188 0.0067 13.12 / 32 / 13 / 13

三因素任务：准确率与格式规范性提升明显

全参数 SFT 在 ATBench 和 R-Judge 上均明显降低无效率；R-Judge 的 F1 下降主要来自基座模型低准确率、高无效率以及偏向 Unsafe 的召回结构带来的指标优势。

三因素维度拆解

SFT 在 Risk Source、Failure Mode、Harm Type 三个维度上都带来准确率提升，其中 Harm Type 的 Macro-F1 提升最明显。

CONCLUSION

总结结论和任务

结论明确：数据质量与全参数 SFT 是当前收益最高的方向

核心结论

  • 微调显著提升二分类和三因素分类准确率，并稳定 JSON 输出格式。
  • 全参数 SFT 的调整能力强于 LoRA，更适合当前小模型任务对齐。
  • R-Judge F1 异常需要结合无效率、召回倾向和计算基数解释，不能单看单项指标。

后续任务

  • 继续扩充和抽检 Reason 标注，降低 LLM 生成标注噪声。
  • 围绕 R-Judge 做类别均衡、阈值校准和错误样本复盘。
  • 增加 Prompt 清洗、去重、数据合并策略的消融实验。
  • 将最优模型接入评估台，形成可复现的训练-评测-展示闭环。
